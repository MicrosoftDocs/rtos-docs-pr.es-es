# <a name="chapter-2---installation-and-use-of-azure-rtos-netx-duo-dns-client"></a>Capítulo 2: Instalación y uso del cliente DNS de Azure RTOS NetX Duo

Este capítulo contiene una descripción de varios problemas relacionados con la instalación, la configuración y el uso del cliente DNS de Azure RTOS NetX Duo.

## <a name="product-distribution"></a>Distribución del producto

El cliente DNS de NetX Duo está disponible en [https://github.com/azure-rtos/netxduo](https://github.com/azure-rtos/netxduo). El paquete incluye dos archivos de origen y un archivo PDF que contiene este documento, como se indica a continuación:

- **nxd_dns.h** Archivo de encabezado para el cliente DNS de NetX Duo
- **nxd_dns.c** Archivo de código fuente C para el cliente DNS de NetX Duo
- **nxd_dns.pdf** Descripción en PDF del cliente DNS de NetX Duo

## <a name="dns-client-installation"></a>Instalación del cliente DNS

Para usar el cliente DNS de NetX Duo, copie los archivos de código fuente *nxd_dns.c* y *nxd_dns.h* en el mismo directorio en el que se ha instalado NetX Duo. Por ejemplo, si NetX Duo se ha instalado en el directorio " *\threadx\arm7\green*", los archivos *nxd_dns.h* y *nxd_dns.c* deben copiarse en este directorio.

## <a name="using-the-dns-client"></a>Uso del cliente DNS

Es fácil usar el cliente DNS de NetX Duo. Básicamente, el código de la aplicación debe incluir *nxd_dns.h* después de incluir *tx_api.h* y *nx_api.h*, con el fin de usar ThreadX y NetX Duo, respectivamente. Después de incluir *nxd_dns.h*, el código de la aplicación puede realizar las llamadas de función DNS especificadas más adelante en esta guía. La aplicación también debe agregar *nxd_dns.c* al proceso de compilación. Este archivo se debe compilar de la misma manera que otros archivos de aplicación y su formulario de objetos se debe vincular junto con los archivos de la aplicación. Esto es todo lo que se necesita para usar DNS de NetX Duo.

> [!NOTE]
> Como DNS usa los servicios de UDP de NetX Duo, debe habilitarse UDP con la llamada a *nx_udp_enable* antes de utilizar DNS.

## <a name="small-example-system-for-netx-duo-dns-client"></a>Sistema de ejemplo pequeño para el cliente DNS de NetX Duo 

El cliente DNS de NetX Duo es compatible con las aplicaciones DNS de NetX existentes. A continuación, se muestra la lista de servicios heredados y su equivalente de NetX Duo:

| Servicio de API de DNS de NetX (solo IPv4) | Servicio de API de DNS de NetX Duo (compatible con IPv4 e IPv6) |
|----------------------------------|----------------------------------------------------|
| nx_dns_host_by_name_get          | nxd_dns_host_by_name_get                           |
| nx_dns_host_by_address_get       | nxd_dns_host_by_address_get                        |
| nx_dns_server_get                | nxd_dns_server_get                                 |
| nx_dns_server_add                | nxd_dns_server_add                                 |
| nx_dns_server_remove             | nxd_dns_server_remove                              |

Consulte la descripción de los servicios de API de cliente DNS de NetX Duo en el capítulo 3 para obtener más detalles.

En el programa de aplicación DNS de ejemplo que se proporciona en esta sección, *nxd_dns.h* se incluye en la línea 6. NX_DNS_CLIENT_USER_CREATE_PACKET_POOL, que permite que la aplicación de cliente DNS cree el grupo de paquetes para el cliente DNS, se declara en las líneas 24 a 26. Este grupo de paquetes se utiliza para asignar paquetes para enviar mensajes DNS. Si se define NX_DNS_CLIENT_USER_CREATE_PACKET_POOL, se crea un grupo de paquetes en las líneas 78 a 98. Si esta opción no está habilitada, el cliente DNS crea su propio grupo de paquetes según la carga del paquete y el tamaño del grupo establecido por los parámetros de configuración en *nxd_dns.h* y descritos en otro lugar de este capítulo.

En las líneas 100 a 112, se crea otro grupo de paquetes para la instancia de IP de cliente que se usa para las operaciones internas de NetX Duo. A continuación, se crea la instancia de IP mediante la llamada a *nx_ip_create* en la línea 115. Es posible que la tarea de IP y el cliente DNS compartan el mismo grupo de paquetes, pero como el cliente DNS suele enviar mensajes más grandes que los paquetes de control enviados por la tarea de IP, si se emplean grupos de paquetes independientes, se usa la memoria de manera más eficaz.

ARP y UDP (que se usa por las redes IPv4) se habilitan en las líneas 129 y 141, respectivamente.

>[!NOTE]
> En esta demostración se usa el controlador "ram" declarado en la línea 44 y utilizado en la llamada a *nx_ip_create*. Este controlador de RAM se distribuye con el código fuente de NetX Duo. Para ejecutar realmente el cliente DNS, la aplicación debe proporcionar un controlador de red físico real para transmitir y recibir paquetes del servidor DNS.

La función de entrada de subproceso de cliente *thread_client_entry* se define debajo de la función *tx_application_define*. Cede inicialmente el control al sistema para permitir que el controlador de red inicialice el subproceso de la tarea de IP.

A continuación, crea el cliente DNS en la línea 257, inicializa la caché de DNS en las líneas 267 a 278 y establece el grupo de paquetes creado previamente en la instancia del cliente DNS en las líneas 281 a 295. Después agrega el servidor DNS en las líneas 297 a 341.

El resto del programa de ejemplo usa los servicios de cliente DNS para realizar consultas de DNS. Las búsquedas de direcciones IP de host se realizan en las líneas 193 y 207. La diferencia entre estos dos servicios, *nxd_dns_host_by_name_get* y *nx_dns_host_by_name_get*, es que el primero guarda los datos de dirección en un tipo de datos NXD_ADDRESS, mientras que el segundo guarda los datos en un tipo de datos ULONG. Además, este último está limitado a las redes IPv4, mientras que el primero se puede usar con redes IPv6 o IPv4. Esto solo es posible si la instancia de IP está habilitada para IPv6. Consulte la guía del usuario de NetX Duo para obtener más detalles sobre cómo habilitar la instancia de IP para la red IPv6.

Otro servicio para las búsquedas de IP de host se muestra en la línea 464, *nx_dns_ipv4_address_by_name_get*. Este servicio difiere de *nx_dns_host_by_name_get* en que devuelve todas las direcciones IPv4 detectadas para el nombre de dominio (o el número máximo que cabe en el búfer proporcionado), no solo la primera dirección recibida en la respuesta del servidor DNS.

Del mismo modo, el servicio *nxd_dns_ipv6_address_by_name_get*, llamado en la línea 380, devuelve todas las direcciones IPv6 detectadas por el cliente DNS, no solo la primera.

Las búsquedas inversas (el nombre de host de la dirección IP) se realizan en la línea 606 (*nx_dns_host_by_address_get*) y de nuevo en las líneas 564 y 588 (*nxd_dns_host_by_address_get*). *nx_dns_host_by_address_get* solo funcionará en redes IPv4, mientras que *nxd_dns_host_by_address_get* funcionará en redes IPv4 o IPv6 (por ejemplo, la instancia de IP está habilitada para redes IPv6 y IPv4).

Dos servicios más para las búsquedas DNS, CNAME y TXT, se muestran en las líneas 627 y 649, respectivamente, para detectar los datos de nombre de dominio y CNAME para el nombre de dominio de entrada. El cliente DNS de NetX Duo es similar a servicios para otros tipos de registro, por ejemplo, MX, NS, SRV y SOA. Consulte el capítulo 3 para obtener descripciones detalladas de todas las búsquedas de tipo de registro disponibles en el cliente DNS de NetX Duo.

Cuando el cliente DNS se elimina en la línea 846 con el servicio *nx_dns_delete*, el grupo de paquetes para el cliente DNS no se elimina, a menos que el cliente DNS haya creado su propio grupo de paquetes. De lo contrario, dependerá de la aplicación la eliminación del grupo de paquetes que ya no se usa.

        return;

        return;

    pointer =  pointer + 2048;

    if (status)
    {

        error_counter++;
        return;
     }

    status =  nx_arp_enable(&client_ip, (void *) pointer, 1024);

    if (status)
    {

        error_counter++;
        return;
     }

        error_counter++;
        return;
     }

## <a name="configuration-options"></a>Opciones de configuración

Hay varias opciones de configuración para compilar el DNS de NetX. Estas opciones se pueden redefinir en *nxd_dns.h.* En la lista siguiente se describe cada una de forma pormenorizada:  

- **NX_DNS_TYPE_OF_SERVICE** Tipo de servicio necesario para las solicitudes UDP de DNS. De forma predeterminada, este valor se define como NX_IP_NORMAL para el servicio de paquetes IP normal.

- **NX_DNS_TIME_TO_LIVE** Especifica el número máximo de enrutadores que un paquete puede pasar antes de que se descarte. El valor predeterminado es 0x80.

- **NX_DNS_FRAGMENT_OPTION** Establece la propiedad de socket para permitir o impedir la fragmentación de los paquetes de salida. El valor predeterminado es NX_DONT_FRAGMENT.

- **NX_DNS_QUEUE_DEPTH** Establece el número máximo de paquetes que se almacenan en la cola de recepción del socket. El valor predeterminado es 5.

- **NX_DNS_MAX_SERVERS** Especifica el número máximo de servidores DNS en la lista de servidor cliente. El valor predeterminado es 5.

- **NX_DNS_MESSAGE_MAX** Tamaño máximo del mensaje DNS para enviar consultas de DNS. El valor predeterminado es 512, que también es el tamaño máximo especificado en el RFC 1035 sección 2.3.4.

- **NX_DNS_PACKET_PAYLOAD_UNALIGNED** Si no se define, el tamaño de la carga del paquete de cliente que incluye los encabezados Ethernet, IP (o IPv6) y UDP más el tamaño de mensaje DNS máximo especificado por NX_DNS_MESSAGE_MAX. Independientemente de si está definido, la carga del paquete es los 4 bytes alineados y almacenados en NX_DNS_PACKET_PAYLOAD.

- **NX_DNS_PACKET_POOL_SIZE** Tamaño del grupo de paquetes de cliente para enviar consultas de DNS si no se define NX_DNS_CLIENT_USER_CREATE_PACKET_POOL. El valor predeterminado es lo suficientemente grande para 16 paquetes de tamaño de carga definido por NX_DNS_PACKET_PAYLOAD y es de 4 bytes alineados.

- **NX_DNS_MAX_RETRIES** Número máximo de veces que el cliente DNS consultará el servidor DNS actual antes de intentar otro servidor o anular la consulta de DNS. El valor predeterminado es 3.

- **NX_DNS_MAX_RETRANS_TIMEOUT** El tiempo de espera máximo de retransmisión en una consulta de DNS a un servidor DNS específico. El valor predeterminado es 64 segundos (64 * NX_IP_PERIODIC_RATE).

- **NX_DNS_IP_GATEWAY_AND_DNS_SERVER** Si se define y la dirección de puerta de enlace IPv4 del cliente es distinta de cero, el cliente DNS establece la puerta de enlace IPv4 como el servidor DNS principal del cliente. El valor predeterminado está deshabilitado.

- **NX_DNS_PACKET_ALLOCATE_TIMEOUT** Establece la opción de tiempo de espera para la asignación de un paquete desde el grupo de paquetes de cliente DNS. El valor predeterminado es 1 segundo (1 * NX_IP_PERIODIC_RATE).

- **NX_DNS_CLIENT_USER_CREATE_PACKET_POOL** Posibilita que el cliente DNS permita a la aplicación crear y establecer el grupo de paquetes de cliente DNS. De forma predeterminada, esta opción está deshabilitada y el cliente DNS crea su propio grupo de paquetes en *nx_dns_create*.

- **NX_DNS_CLIENT_CLEAR_QUEUE** Permite que el cliente DNS borre los mensajes DNS antiguos de la cola de recepción antes de enviar una nueva consulta. La eliminación de estos paquetes de consultas de DNS anteriores impide que la cola del socket de cliente DNS se desborde y quite paquetes válidos.

- **NX_DNS_ENABLE_EXTENDED_RR_TYPES** Permite que el cliente DNS consulte otros tipos de registro DNS (por ejemplo, CNAME, NS, MX, SOA, SRV y TXT).

- **NX_DNS_CACHE_ENABLE** Permite que el cliente DNS almacene los registros de respuesta en la memoria caché de DNS.
